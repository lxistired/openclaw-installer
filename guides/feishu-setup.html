飞书机器人创建教程

6 步完成飞书 AI 机器人配置 — 包含权限批量导入 & 配对指南

1

创建飞书应用 + 添加机器人能力

在另一个浏览器窗口中，打开飞书开放平台：

https://open.feishu.cn/app

使用你的飞书账号登录。如果没有飞书账号，需要先注册一个。

任何飞书账号都可以创建应用。如果登录后提示需要创建团队，直接填一个团队名称即可（比如「我的工作室」），无需企业认证。

登录后，在「我的应用」页面：

1. 点击右上角的 「创建应用」 按钮

2. 选择 「企业自建应用」（即使是个人使用也选这个）

3. 填写应用信息：

  • 应用名称：填写你想要的名字，如「我的AI助手」
  • 应用描述：填写简单描述，如「基于 OpenClaw 的 AI 助手」

4. 点击 「创建」

创建完成后，立即添加机器人能力：

1. 在应用详情页的左侧菜单中，点击 「应用能力」

2. 点击 「添加应用能力」

3. 找到 「机器人」，点击 「添加」

启用机器人能力后，你的应用就可以在飞书群聊和私聊中被 @ 或直接对话了。
2

批量导入权限（一键操作）

在左侧菜单中点击 「权限管理」，然后点击右上角的 「批量开通」 按钮。

将下方 JSON 复制并粘贴到输入框中，点击 「确认」 即可一次性开通所有权限：

{ "scopes": { "tenant": [ "contact:user.employee_id:readonly", "contact:user.base:readonly", "im:chat.access_event.bot_p2p_chat:read", "im:chat.members:bot_access", "im:message", "im:message.group_at_msg:readonly", "im:message.p2p_msg:readonly", "im:message:readonly", "im:message:send_as_bot", "im:message.reactions:read", "im:resource" ], "user": [ "im:chat.access_event.bot_p2p_chat:read" ] } }
使用「批量开通」比逐个搜索权限快很多，且不会遗漏。如果你的飞书开放平台版本没有「批量开通」按钮，也可以在搜索框中逐个搜索上述权限名称并开通。
权限说明（点击展开）
  • contact:user.employee_id:readonly — 读取用户工号
  • contact:user.base:readonly — 获取用户基本信息（显示发送者姓名）
  • im:chat.access_event.bot_p2p_chat:read — 接收私聊事件
  • im:chat.members:bot_access — 获取群成员信息
  • im:message — 获取与发送消息
  • im:message.group_at_msg:readonly — 接收群聊 @机器人的消息
  • im:message.p2p_msg:readonly — 接收用户私聊消息
  • im:message:readonly — 读取消息内容
  • im:message:send_as_bot — 以机器人身份发送消息
  • im:message.reactions:read — 读取消息表情回复
  • im:resource — 访问消息中的图片/文件等资源
3

复制 App ID 和 App Secret

在左侧菜单点击 「凭证与基础信息」，可以看到：

App ID：cli_xxxxxxxxxxxxxxxx
App Secret：xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
请立即复制并保存这两个值！稍后需要粘贴到安装程序中。

点击 App Secret 旁边的「显示」按钮查看完整密钥，然后点击「复制」。

4

配置事件订阅（WebSocket + 接收消息）

这是最关键的一步——不配置事件订阅，机器人将无法接收任何消息！

在左侧菜单中点击 「事件与回调」：

第一步：选择接收方式

1. 在「事件配置」页面，点击订阅方式旁的 「编辑」

2. 选择 「使用长连接接收事件」（即 WebSocket 模式）

3. 点击 「保存」

选择「长连接」模式无需公网 IP 和域名，OpenClaw 会通过 WebSocket 主动连接飞书服务器。

第二步：添加事件

1. 点击 「添加事件」 按钮

2. 搜索 im.message.receive_v1

3. 找到 「接收消息」 事件，点击添加

4. 点击 「保存」

必须添加 im.message.receive_v1 事件！没有这个事件订阅，机器人虽然在线但不会响应任何消息。
5

发布应用

配置完权限和事件后，需要发布应用才能生效：

1. 点击左侧菜单的 「版本管理与发布」

2. 点击 「创建版本」

3. 填写版本号（如 1.0.0）和更新说明

4. 设置可用范围（建议选择「全部成员」或指定人员）

5. 点击 「提交审核」

提交后需要团队管理员审批。如果你自己就是管理员（个人创建的团队默认就是），可以直接在飞书管理后台审批通过。
个人创建的团队，你自己就是管理员，打开飞书 →「管理后台」→「应用审核」直接通过即可。
6

配对（Pairing）— 首次连接必须

应用发布并审批通过后，还需要完成一次设备配对才能让机器人响应消息：

操作步骤：

1. 回到安装程序窗口，完成配置后启动 OpenClaw：

openclaw gateway

2. 在飞书中找到你的机器人（搜索你设置的机器人名称），发送一条任意消息

3. 机器人会回复一个 6 位数的配对码 (Pairing Code)

4. 在终端中运行配对命令：

openclaw pairing approve feishu <配对码>

例如：openclaw pairing approve feishu 123456

5. 配对成功后，再次发送消息，机器人就会正常响应了！

配对只需要做一次！配对成功后的信息会被保存，后续重启 gateway 不需要重新配对。
也可以通过「配置飞书.bat → 选项 5」完成配对操作。

❓ 常见问题

Q: 找不到「创建应用」按钮？

A: 登录飞书开放平台后，如果还没有团队，系统会先引导你创建一个团队。随便填个团队名称（如「我的工作室」）即可，不需要企业认证。创建完团队后就能看到「创建应用」按钮了。


Q: 没有「批量开通」按钮怎么办？

A: 部分飞书开放平台版本可能没有批量导入功能。此时需要在「权限管理」→「API 权限」标签页下，逐个搜索权限名称并开通。展开「消息与群组」和「通讯录」分类可以更快找到。


Q: 审批一直没通过？

A: 如果是个人创建的团队，你自己就是管理员，打开飞书 →「管理后台」→「应用审核」直接通过即可。


Q: 机器人添加到群后没反应？

A: 请逐项排查：

  1. 确认事件订阅方式选择了「长连接」(WebSocket)（步骤 4）
  2. 确认已添加 im.message.receive_v1 事件（步骤 4）
  3. 确认所有权限已开通（步骤 2）
  4. 确认应用已审批通过（步骤 5）
  5. 确认已完成设备配对 Pairing（步骤 6）
  6. 确认 OpenClaw Gateway 正在运行（openclaw gateway）

Q: WebSocket 连接不上 / gateway 启动后没有收到消息？

A: 常见原因：

  1. 未选择「长连接」方式 — 请在「事件与回调」中确认订阅方式为 WebSocket
  2. 未添加 im.message.receive_v1 事件 — 必须添加此事件才能接收消息
  3. 未完成配对 — 首次使用必须完成 Pairing（步骤 6）
  4. 网络问题 — 运行 配置飞书.bat → 选项 3 进行全面诊断

Q: 什么是 Pairing（配对）？

A: Pairing 是 OpenClaw 的安全机制，用于将飞书机器人与本地 OpenClaw 实例绑定。首次连接时，机器人会发送一个 6 位配对码，你需要在终端中确认配对。这确保了只有授权的设备可以控制机器人。配对只需做一次，信息会持久化保存。
